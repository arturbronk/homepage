W03E03 - HTML II

Znacznik <strong> podkreśla istotność i poważność frazy w nim zawartej.

Znacznik <em> podkreśla inne znaczenie tekstu np. ironię

Znaczniki <b> i <i> nie mają znaczenia semantycznego tzn. są tylko wyróżnikami

<a href="target">opis łącza</a>

Link wewnętrzny - podajemy atrybut id do sekcji np.

<section id ="notatki">
<a href="#notatki">Czytaj notatki</a>

Znacznik <nav> znacznik nawigacji, który umieszczamy wraz z znacznikiem <header> nad znacznikiem <main> dla nagłówka, który będzie powtarzalny na podstronach.

To samo tyczy się znacznika <footer> , który wyrzucamy poza <main>

<img src="plik" alt="Tekst alternatywny"/> - jest to znacznik samodomykający się

Podsumowanie

  • Atrybuty są "niewidzialne" - chowamy je w znaczniku
  • Nadają elementom dodatkowe właściwości (href, src)
  • Pozwalają na grupowanie elementów (class)
  • Pozwalają na identyfikowanie elementów (id)
  • Można je miksować
  • Spis atrybutów "globalnych"

W03E05 - CSS II

Style inline, klasy, selektor id & łączenie selektorów

Styl przy znaczniku z wykorzystaniem atrybutu style nazywamy stylem inline

<znacznik style="color:yellow">treść</znacznik>

Klasa - .() - selektor klasy. Odwołujemy się do niej podając nazwę klasy po kropce.

<znacznik class="name" >treść</znacznik>

.name {
właściwość: wartość;
}

Przykład:

.duza-czcionka {
font-size: 60px;
}

ID - #(hash) - selektor id. Odwołujemy się podając nazwę po #

<znacznik id="name" >treść</znacznik>

#name {
właściwość: wartość;
}

Przykład:

#moje-id-znacznika {
właściwość: wartość;
}

Łączenie selektorów - przykład

nav a {
color: #fff333;
}

Dotyczy linków w sekcji parent czyli <nav> jak i wyżej np. header nav a

!important; nadpisuje wszystkie inne style

Specyficzność CSS

Maciek rekomenduję dodawać klasy do wszystkich elementów i unikać zagnieżdżeń

Nazewnictwo klas

  • używaj pojedynczych słów z j. angielskiego
  • nazywaj rzeczy zgodnie z ich funkcją na stronie
  • startuj z małej litery np. header nie Header
  • nie zaczynaj od cyfry
  • jeśli musisz użyć dwóch słów użyj tzw. kebab case np. news-listing
  • staraj się, żeby element miał jedną klasę (nie zwiększaj specyficzności bez potrzeby)

Podsumowanie

  • html class="name" = css .name
  • html id="name" = css #name
  • stylujemy po klasach (bez zagnieżdżeń!)
  • nazywamy klasy po angielsku zgodnie z przeznaczeniem
  • tworzymy klasy nawet mamy jeden element danego typu
  • specyficzność może dać popalić ;)
  • nie używamy !important i stylów inline'owych
